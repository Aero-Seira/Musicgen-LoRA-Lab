# 基于 MusicGen-Melody 的音乐音频续写与 LoRA 泛化能力研究方案

## 1. 项目概述

### 1.1 项目题目

基于 MusicGen-Melody 的音乐音频续写与 LoRA 泛化能力研究

### 1.2 核心任务

本项目研究音乐音频续写任务，即给定一段音乐的前半部分，让模型生成后续片段。

- 输入：音频前 `20s`
- 输出：生成后续 `10s`
- 主数据集：GTZAN
- 基座模型：MusicGen-Melody
- 微调方法：LoRA 参数高效微调
- 对比对象：
  - 预训练 MusicGen-Melody
  - LoRA 微调后的 MusicGen-Melody

### 1.3 项目目标

1. 构建一个完整的“前 20 秒生成后 10 秒”的音乐续写流程。
2. 使用 GTZAN 数据集对 MusicGen-Melody 进行 LoRA 微调。
3. 比较预训练模型与 LoRA 微调模型在 GTZAN 测试集上的表现。
4. 使用外部音频测试微调模型的泛化能力。
5. 结合客观指标和主观听测分析生成结果。

## 4. 数据处理方案

### 4.1 数据集

主数据集使用 GTZAN：

- 共约 1000 条音频
- 每条约 30 秒
- 包含 10 种音乐流派
- 常见类别包括 blues、classical、country、disco、hiphop、jazz、metal、pop、reggae、rock

### 4.2 数据划分

建议按比例划分：

| 数据集 | 比例 | 用途 |
| --- | ---: | --- |
| 训练集 | 70% | LoRA 微调 |
| 验证集 | 15% | 调参和早停 |
| 测试集 | 15% | 最终评估 |

如果时间有限，可以先做小样本实验：

- 每个流派抽取 `20-30` 条音频。
- 先跑通完整闭环。
- 再逐步扩大数据量。

### 4.3 音频预处理

每条音频统一处理为：

- 单声道
- 统一采样率
- 固定长度 `30s`
- 音量归一化
- 保存为 `.wav`

注意：MusicGen 内部通常使用特定采样率。预处理阶段可以先统一到 `16kHz`，实际训练和推理时再根据模型要求调整到对应采样率。

### 4.4 样本切分

每条 30 秒音频切分为：

| 片段 | 时间范围 | 用途 |
| --- | --- | --- |
| 输入音频 | `0s-20s` | 模型条件输入 |
| 目标音频 | `20s-30s` | 参考续写片段 |

## 5. 实验设计

### 5.1 实验一：预训练模型基线

目的：建立未经微调的 MusicGen-Melody 基线性能。

设置：

- 模型：预训练 MusicGen-Melody
- 数据：GTZAN 测试集
- 输入：前 `20s`
- 输出：生成后续 `10s`

结果保存：

```text
outputs/baseline/
```

### 5.2 实验二：LoRA 微调模型

目的：验证 LoRA 微调是否能提升模型对 GTZAN 音乐风格的续写能力。

设置：

- 模型：MusicGen-Melody + LoRA Adapter
- 训练数据：GTZAN 训练集
- 测试数据：GTZAN 测试集
- 输入：前 `20s`
- 输出：生成后续 `10s`

推荐初始超参数：

| 参数 | 建议值 |
| --- | --- |
| LoRA rank | `8` 或 `16` |
| LoRA alpha | `16` 或 `32` |
| learning rate | `1e-4` 到 `5e-5` |
| batch size | `1-4` |
| epoch | `5-10` |
| precision | mixed precision |

结果保存：

```text
outputs/lora/
```

### 5.3 实验三：外部音频泛化测试

目的：测试 LoRA 微调模型面对未见过音频时的泛化能力和鲁棒性。

外部音频来源：

| 来源 | 示例 |
| --- | --- |
| 网络音频 | YouTube、Bilibili、Free Music Archive |
| 自录音频 | 哼唱、钢琴、吉他、环境声 |
| 风格近似音频 | 流行、摇滚、古典、爵士等 |

测试流程：

1. 收集 `10-20` 条外部音频。
2. 每条音频裁剪为 `30s`。
3. 取前 `20s` 作为模型输入。
4. 分别使用预训练模型和 LoRA 微调模型生成后续 `10s`。
5. 保存生成结果并进行主观听测。

结果保存：

```text
outputs/external/
```

## 6. 评估方案

### 6.1 客观评估指标

推荐采用轻量可实现的指标，保证项目能够完整落地。

| 指标 | 作用 | 解释 |
| --- | --- | --- |
| Mel-spectrogram MSE | 频谱差异 | 比较生成后 10 秒与真实后 10 秒的频谱距离 |
| Chroma Similarity | 和声/旋律相似度 | 评估生成片段与真实片段在音高结构上的接近程度 |
| Transition Smoothness | 过渡平滑性 | 比较输入结尾和生成开头的频谱变化 |
| FAD | 整体音频质量 | 条件允许时使用，数值越低越好 |

如果 FAD 实现成本过高，可以在报告中说明资源限制，并使用 Mel 频谱、Chroma 和主观评测作为主要评价依据。

### 6.2 主观听测指标

邀请 `5-10` 名同学或志愿者进行主观评分。

评分范围：`1-5` 分。

| 维度 | 含义 |
| --- | --- |
| 连贯性 Coherence | 生成片段与前 20 秒是否自然衔接 |
| 风格一致性 Style Consistency | 生成片段是否保持输入音乐的风格 |
| 音质 Overall Quality | 是否有明显噪声、断裂或失真 |
| 总体偏好 Preference | 预训练模型与 LoRA 模型二选一 |

主观评测建议采用盲听方式，即不告诉评审样本来自哪个模型。

## 8. 推荐实现顺序

### 第一步：先跑通基线

优先完成：

1. 安装环境。
2. 下载或整理 GTZAN。
3. 写音频预处理脚本。
4. 加载 MusicGen-Melody。
5. 使用测试集前 `20s` 生成后 `10s`。
6. 保存基线生成结果。

这一阶段完成后，即使后续 LoRA 训练遇到问题，也已经有可展示的模型生成结果。

### 第二步：整理 LoRA 训练数据

完成：

1. 将 GTZAN 划分为训练集、验证集、测试集。
2. 统一音频格式。
3. 生成训练清单文件。
4. 检查每条音频长度和采样率。

### 第三步：LoRA 小样本微调

先用小样本验证训练流程：

- 每个流派选少量样本。
- 训练少量 epoch。
- 检查 loss 是否下降。
- 检查生成音频是否可听。

### 第四步：扩大实验

在流程跑通后：

1. 增加训练数据。
2. 调整 LoRA rank、learning rate、epoch。
3. 生成完整测试集结果。
4. 保存模型权重和配置文件。

### 第五步：泛化测试

完成外部音频实验：

1. 收集外部音频。
2. 统一预处理。
3. 使用预训练模型和 LoRA 模型分别生成。
4. 保存对比样本。
5. 分析成功和失败案例。

### 第六步：评估与报告

完成：

1. 计算客观指标。
2. 组织主观听测。
3. 汇总评分结果。
4. 绘制表格和图。
5. 撰写最终报告。

## 11. 风险与应对策略

| 风险 | 表现 | 应对策略 |
| --- | --- | --- |
| 显存不足 | 1.5B 模型无法训练 | 先用小模型跑通流程，开启混合精度，减小 batch size |
| LoRA 效果不明显 | 生成结果提升有限 | 调整 rank、learning rate、epoch，增加高质量训练样本 |
| 外部音频效果差 | 泛化结果不稳定 | 分析音频风格差异，选择风格更接近 GTZAN 的外部样本 |
| FAD 实现复杂 | 指标计算耗时或依赖复杂 | 使用 Mel MSE、Chroma Similarity 和主观听测替代 |
| 训练时间过长 | 无法在作业周期内完成 | 先做小样本实验，保证完整闭环，再扩大实验 |
| 音频续写不连贯 | 20s 到生成片段衔接突兀 | 增加过渡平滑性分析，调整生成参数 |

## 13. 推荐结论表达

报告中可以围绕以下结论展开：

1. MusicGen-Melody 具备一定的音频续写能力，但未经微调时对 GTZAN 特定流派的风格保持不稳定。
2. LoRA 微调可以在较低算力成本下增强模型对训练数据音乐风格的适应能力。
3. 在 GTZAN 测试集上，LoRA 模型相比预训练模型可能在风格一致性和主观连贯性上有所提升。
4. 在外部音频上，LoRA 模型的泛化能力取决于外部音频与训练数据的风格接近程度。
5. 资源受限条件下，LoRA 是一种较实用的音乐生成模型适配方法。

## 14. 推荐执行策略

本项目最稳妥的执行策略是：

```text
先用小模型和小样本跑通完整闭环
        |
        v
确认数据、训练、生成、评估都可运行
        |
        v
逐步扩大数据量和模型规模
        |
        v
补充外部音频泛化实验
        |

```

不要一开始就把目标锁死在 `MusicGen-Melody 1.5B + 完整 GTZAN + 完整 FAD`。优先完成可运行、可展示、可分析的完整项目，再根据剩余时间和算力升级实验规模。
